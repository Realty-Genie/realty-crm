---
title: Add Members
description: Add users to a workspace
---

Add one or more users to a workspace as members.

<EndpointBadge
  method="POST"
  path="/api/v1/memberships/add-members"
  auth={true}
/>

## Request Headers

```http
Authorization: Bearer <access_token>
```

## Request Body

```json
{
  "workspaceId": "string",
  "users": ["user_id_1", "user_id_2"]
}
```

## Parameters

| Field         | Type     | Required | Description                      |
| ------------- | -------- | -------- | -------------------------------- |
| `workspaceId` | string   | Yes      | ID of the workspace              |
| `users`       | string[] | Yes      | Array of user IDs to add (min 1) |

## Response

Returns `201 Created` on success.

```json
{
  "message": "Members added successfully",
  "memberships": [
    {
      "_id": "507f1f77bcf86cd799439011",
      "workspace": "507f1f77bcf86cd799439012",
      "user": {
        "name": "Jane Doe",
        "email": "jane@example.com"
      },
      "role": "AGENT",
      "isAccepted": false,
      "isRemoved": false,
      "createdAt": "2024-01-01T00:00:00.000Z"
    }
  ]
}
```

New members are added with `isAccepted: false` and must accept the invitation.

## Default Role

New members are added with the `AGENT` role by default. Use the update endpoint to change their role.
